Setting up Rails application with Mina on all distros, Ruby, Rails

Deploy your Rails application with Mina - ALL DISTROS

Difficulty: 4
Time: 1 hr

In this article, we will assist you to understand the installation of Mina and deployment of Rails application with MINA on all distros. We assume that you are logged in as a sudo user.

Understanding MINA

Mina is a Ruby based tool used for creating scripts with a list of tasks to perform, grouped under a name, and to facilitate everyday operations. Mina converts these RAKE based programming files to bash scripts, connects to a defined remote server, and executes them.

Prepare the system

  1. Update the default tools.
    sudo yum install update //For CentOS,Fedora
    sudo apt-get install update //For Ubuntu,Debian
    sudo paceman -Syu //For Arch linux
  2. Install the development tool
    sudo yum groupinstall 'Development-tools'//For CentOS,Fedora
    sudo apt-get install build-essential //For Ubuntu,Debian
    sudo pacman -Sy base-devel //For Arch linux

Create a Rails application

  1. Create a new directory and give the following permission.
    sudo mkdir -p /var/www/
    sudo chmod 0777 -R /var/www/
  2. Go to the above directory and create a Rails application.
    cd /var/www
    rails new myapp

Install MINA

  1. Go to Rails application directory and install Mina using following command.
    Cd /var/www/myapp
    gem install mina
  2. Create deploy.rb using following command.
    Mini init
  3. Create a user for deployment
    sudo useradd username
    sudo passwd username
  4. Now open /etc/sudoers file
    sudo vim /etc/sudoers
  5. Scroll down to the bottom of this file. Then add the following line just after where the root is defined. 
    username ALL=(ALL) ALL //For Arch Linux,CentOS,Fedora
    username ALL=(ALL) ALL //For Arch Linux,CentOS,Fedora
  6. Save and exit

Create a Git Repository

In this section, we are going to make a repository on GitHub. So login on GitHub and create a myapp repository.

  1. Install Git
    sudo pacman -S git //For Arch Linux
    sudo apt-get install git //For Ubuntu,Debian
    sudo yum install git //For centOS,Fedora
  2. Initiate the repository
    git init
    git add .
    git config --global user.name "GitHub_UserName"
    git config --global user.email "GitHub_Email_Address"
  3. Commit the changes
    git commit -m "first commit"
  4. Add your GitHub repository link
    git remote add origin git@github.com: GitHub_UserName/myapp.git
  5. Create an RSA/SSH key
    ssh-keygen -t rsa
    sudo cat /root/.ssh/id_rsa.pub

    Or

    sudo cat /home/username/.ssh/id_rsa.pub //for sudo user
  6. Push the project's source code to your Github account
    git push -u origin master
  7. Change the ownership of rails application to the deployer.
    sudo chown -R deployer_username /var/www/myapp

Deployment of Rail application

  1. Go to the application directory
    cd /var/www/myapp
  2. Open deploy.rb file
    sudo vim config/deploy.rb
  3. Mina requires bundler, rails, git and ruby, so add the following lines in the beginning of this file. 
    require 'mina/bundler'
    require 'mina/rails' 
     require 'mina/git' 
    require 'mina/rbenv'
    # require 'mina/rvm'(uncomment this if you have installed ruby with rvm)
  4. Set the domain name or IP and then set the path for the Rails application folder, repository and branch. 
     set :rails_env, 'production'
 set :domain,     'ip_adress' //your public IP
 set :deploy_to,  "/home/Deployer_username/app/#{rails_env}"
 set :app_path,   "#{deploy_to}/#{current_path}" //rails application path
set :repository, 'https://github.com/GitHub_username/mynewapp.git'
    set :branch,     'master'
  5. In the environment task section, uncomment line which you want to choose. 
     task :environment do  
     invoke :'rbenv:load'
     # invoke :'rvm:use[ruby-1.9.3-p125@default]'   
    End
  6. 6. Add the following lines to clone the repository. Then configure the shared paths, execute a bundle to set all the required gems to the server. Optionally you can migrate the database and precompile the assets. 
    task :deploy => :environment do 
     deploy do 
       invoke :'git:clone' 
        invoke :'deploy:link_shared_paths' 
        invoke :'bundle:install' 
        
        #invoke :'rails:db_migrate' 
        #invoke :'rails:assets_precompile' 
       
        to :launch do 
          queue "touch #{deploy_to}/tmp/restart.txt" 
        end 
      end 
   end
  7. Save and exit.
  8. Run the following command to create the folder structure in deploy.rb file.
    mina setup -verbose
  9. Create all Folders and deploy Rails Application using the following commands.
    mina setup
    mina deploy
  10. Test your application by running the following command in the application directory:
    bundle exec rails server
  11. For binding to web browser, run the following command:
    rails server --binding=[your droplet's IP]

Next Steps

By using all the above mentioned steps, you can easily deploy your Rails application with mina on all distros.
