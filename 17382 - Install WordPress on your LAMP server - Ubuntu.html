Installing Wordpress on a LAMP server for Ubuntu.

Install WordPress on your LAMP server – Ubuntu

Difficulty: 1
Time: 15 minutes

WordPress has become much more than its humble beginnings in 2003 as a self-hosted blogging tool; it is the Internet’s leading content management system (CMS). This open-source CMS powers all types of sites, including major brands such as The New Yorker, Variety, TechCrunch, and Best Buy. With a MySQL™ and PHP™ architecture, WordPress is easy to install and has endless options for website configuration thanks to tens of thousands of plug-ins and themes from its rich and active community.

This tutorial walks you through setting up WordPress on an Ubuntu® (14.04, 14.10) system. The steps will be for a non-root user with sudo, or administrative privileges.

Prerequisites

Make sure you that have a non-root user with sudo (administrator) privileges set up on your Ubuntu server. You’ll also need a Linux, Apache, MySQL, and PHP (LAMP) stack configured. Make sure Apache, MySQL, and PHP are up and running. For a how-to on installing a LAMP stack, see Build a LAMP stack (Linux, Apache, MySQL, PHP) – Ubuntu.

As you install the software, make note of your MySQL root account (administrator) password. You’ll need it for these tasks.

Create a MySQL database and administrator for WordPress

Set up your MySQL database with an administrator so that WordPress has a place to store all your site and user information.

Remember: All MySQL code statements must end with a semi-colon (;).

  1. Log into your MySQL admin account.
    mysql -u root -p
  2. At the prompt, enter your MySQL root password.
  3. At the MySQL prompt, create your WordPress database. For this tutorial, we’ll use the database name fittingly as “wordpress.”
    CREATE DATABASE wordpress;
  4. Create the database administrator account and set the user’s credentials. For this tutorial, we’ll use “wordpressadmin” as the example user name, and “password” as the password.
    CREATE USER wordpressadmin@localhost IDENTIFIED BY 'password';

    As a good security practice, create a strong password. Keep your username and password handy, as you’ll need them later.

  5. Give, or grant, your admin permissions to the database.
    GRANT ALL PRIVILEGES ON wordpress.* TO wordpressadmin@localhost;
    GRANT is one of the account management statements the MySQL server notices and loads into memory right away. So a manual reload/restart should not be necessary. But if you’d like, you can check the permissions using SHOW GRANTS.
    SHOW GRANTS FOR 'wordpressadmin'@'localhost';

  6. If for some reason the privileges didn’t take, flush the privileges to activate the user. Flush forces a reload of the cache.

    FLUSH PRIVILEGES;
  7. You're finished! Exit MySQL to bring you back to your normal command prompt.
    exit

Download WordPress

Your system is now ready to install WordPress into your directory. WordPress is available in a tar.gz or zip format. We will be getting the tar.gz file.
  1. Download the WordPress files from the official site using wget.
    cd ~
    wget http://wordpress.org/latest.tar.gz

    This URL will always link to a compressed file of the latest stable version of WordPress.

  2. Extract, or untar, the application directory.
    tar xzvf latest.tar.gz

    This command creates a directory called wordpress in your home directory.

Configure WordPress

  1. Go into the WordPress directory that you just unpacked.
    cd ~/wordpress
  2. Edit the following lines with the values in wp-config.php.
    cp wp-config-sample.php wp-config.php
  3. Open the config file in your text editor.
    vim wp-config.php
  4. Look for this section of the code below (starting with MySQL settings…) and enter your corresponding values of the three DB_ variables (database name, database username, mysql root password) with the credentials you just created. Use the credentials that you just created.
    // ** MySQL settings - You can get this info from your web host ** //
    /** The name of the database for WordPress */
    define('DB_NAME', 'wordpress');
    /** MySQL database username */
    define('DB_USER', 'wordpressadmin');
    /** MySQL database password */
    define('DB_PASSWORD', 'password');
  5. Save and close.

Copy files to document root

After you’ve configured WordPress, transfer the files to Apache’s root directory so that the application can serve content to your site visitors.

  1. Transfer your WordPress files using rsync, a reliable file transfer and synchronization program.
    sudo rsync -avP ~/wordpress/ /var/www/html/
    This command safely copies the contents from the directory that you unpacked earlier to the document root.
  2. Go to the document root to get back ownership for the files in /var/www/html.
    cd /var/www/html

  3. For security purposes, change ownership to a sudo non-root user to interact with your site. This can be your regular user, or an additional user you create.

    For this tutorial, we’ll work with an example account that we’ll assume is already created called "wordpress_user."

    Give your web server process, www-data, ownership values to allow Apache to interact with the content:

    sudo chown -R wordpress_user:www-data *

Set directory permissions

Open the permissions of your uploads directory so that you can upload images (as the name suggests) and other content to your site. The default permissions are too limiting.

  1. Create the uploads directory.
    mkdir /var/www/html/wp-content/uploads
    This will be the parent directory.
  2. Apache needs to be able to write to the directory in order to save files in it. Change the ownership of the uploads directory to a group ownership.
    sudo chown -R :www-data /var/www/html/wp-content/uploads

    This allows the web server to create files and directories under the directory, which means you’ll be able to upload content to the server.

Install WordPress

Good news! The rest of your installation will be done through the WordPress web interface.

  1. Enter your server’s domain name or public IP address in your web browser. http://server_domain_name_or_IP

  2. At the WordPress configuration page, fill out your information to create your administrator account.

  3. WordPress confirms your installation by displaying your username and password.

  4. Click Log In. WordPress prompts you to log in with your new credentials.

All done! You’ll be taken to the dashboard of your new site.
