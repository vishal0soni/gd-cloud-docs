Setting up Ruby on Rails using rbenv in Fedora, Ruby, Rails

Install Ruby on Rails using rbenv – Fedora

Difficulty: 1
Time: 30 minutes

Ruby is a pure object oriented language like PERL and Python and Rails is a framework for web application development written in Ruby. In this article, you will learn to install Ruby on Rails on Fedora, with the help of rbenv. rbenv is a new lightweight Ruby version management tool

Note: User should have sudo privileges to run the following command.

Prerequisites

To setup the Fedora server, please refer 17322 - Initial Server Setup - Fedora link. Optionally, you can refer 17367 - Install Git - Fedora link to install Git.

Install rbenv

  1. Before installing Ruby you need to install the dependencies of rbenv and Ruby with yum with the help of
    sudo yum install -y git-core zlib zlib-devel gcc-c++ patch readline readline-devel libyaml-devel libffi-devel openssl-devel make bzip2 autoconf automake libtool bison curl sqlite-devel
  2. Install rbenv and check for rbenv into .rbenv directory
    cd
    git clone git://github.com/sstephenson/rbenv.git .rbenv

  3. Add a directory to your $PATH for accessing the rbenv

    echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bash_profile

  4. Add rbenv init in to the shell for enabling the shims

    echo 'eval "$(rbenv init -)"' >> ~/.bash_profile
    exec $SHELL
  5. These commands will install the latest development version of Ruby-build into the ~/.rbenv/plugins/ruby-build directory
    git clone git://github.com/sstephenson/ruby- build.git~/.rbenv/plugins/ruby-build
  6. Add a directory to your $PATH for accessing Ruby
    echo 'export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH"' >> ~/.bash_profile
    exec $SHELL
  7. This installs rbenv into your home directory and sets the appropriate environment variables that will allow rbenv to the active version of Ruby

Install Ruby

Before you start using rbenv, make sure which version of Ruby you want to install. In this article we are installing the latest version of Ruby.

  1. Install the dependency of Rails, before you install Ruby by typing
    rbenv install -v latest_version
  2. The below command will set the default version of Ruby that all of your shells will use
    rbenv global latest_version
  3. The below command will display the version of Ruby that was installed
    ruby -v

Install RubyGems

  1. When you install gems, by default documentation files will be installed. But it takes time, so here we first stop the document installation by disabling them
    echo "gem: --no-document" > ~/.gemrc
  2. After the document step is disabled, install the bundler gem. The Bundler gem is an essential tool for managing gems when developing and running Rails applications
    gem install bundler

Setup Rails

  1. To install Rails, type
    gem install rails

    This will take some time, so be patient. Once completed, Ruby on Rails will be installed.

  2. To install Shims for all Ruby executables known to rbenv, use sub-command rehash
    rbenv rehash
  3. Find out the version of both Ruby and Rails
    ruby -v
    rails –v

Install JavaScript Runtime

A few Rails features, such as the Asset Pipeline, depend on a Javascript runtime. Install Node.js to provide this functionality.
  1. Add Fedora's Extra Packages for Enterprise Linux (EPEL) yum repository to provide the ability to install software packages which are not included in the official Fedora or Red Hat Enterprise Linux (RHEL) repositories
    sudo yum -y install epel-release
  2. After you have installed the EPEL release, install the Node.js package
    sudo yum install nodejs
