This article walks thorugh setting up basic HTTP authentication in Apache using .htaccess

Set up HTTP Authentication with Apache - Debian

Difficulty: 2
Time: 10 minutes

In this article, you will learn to protect your directories or websites using HTTP authentication and the the .htaccess file. In the following example, we will require user authentication in order to view the Apache® test page.

Prerequisite

You must have Initial Server Setup done on your server, and installed Apache.

  • If you haven't installed Apache yet, run these commands to install it.
    sudo apt-get update
    sudo apt-get install apache2

Create .htaccess file

.htaccess stands for Hypertext Access. This file contains the authentication details.

  1. Create the .htaccess file under Document Root directory to apply the authentication to the entire site or a particular directory.
    sudo vim /var/www/.htaccess
  2. Type in the following block of text: 
    AuthType Basic
AuthName "Restricted. Only authorized users."
AuthUserFile "/etc/apache2/.htpasswd"
Require valid-user
  3. Save and exit the file.
  4. Below is an explanation of what each of these directives mean:
    Directive Description
    AuthType Basic Specifies the kind of authentication that is being implemented. "Basic" implies that it uses a password file to implement the password authentication.
    AuthName The message that is displayed at the password prompt. Set it to anything you want.
    AuthUserFileThe location where you would store the password file.
    Require valid-userIndicates that only users who are stored in the password file will be granted access to the phpMyAdmin login screen.

Create user

  1. To generate user/passwords, you need the htpasswd tool. Prior to using the htpasswd, you need to install Apache utilities.
    sudo apt-get update
    sudo apt-get install apache2-utils
  2. Create the .htpasswd file in the path mentioned in the previous section along with the user name. You will be prompted to set and confirm a password for the user.
    sudo htpasswd -c /etc/apache2/.htpasswd newuser1

Update Apache configuration file

  1. Open the Apache configuration file.
    sudo vim /etc/apache2/sites-available/default
  2. You need to include the AllowOverride AuthConfig directive in the Apache configuration file under the document root directory. In this case, it is /var/www. The <Directory> section looks something like: 
    <Directory /var/www/>
    Options Indexes FollowSymLinks
    AllowOverride AuthConfig
    Order allow,deny
    Allow from all
</Directory>
  3. Save and exit the file.
  4. Restart Apache.
    sudo service apache2 restart
  5. If you navigate to http://server-ip-address/ on your web browser. You will be challenged by a prompt asking for a valid username and password. Enter the newly created user newuser1 and the password you set. After authentication you should see the Apache test page.

Conclusion

You have successfully secured your website using HTTP authentication.
