Configures an Nginx server as an SSL proxy for a Jenkins server.

Configure Nginx with SSL as a Reverse Proxy for Jenkins - Debian, Ubuntu

Difficulty: 3
Time: 30 Minutes

Jenkins is a popular tool for continuous pipeline integration, offering features for managing and monitoring project build jobs. This article describes how to use NGINX as a reverse proxy with SSL for Jenkins.  It assumes you have already installed Nginx and  acquired and installed an SSL certificate. Although the steps for creating a reverse proxy NGINX configuration with SSL for Jenkins are covered here, you may wish to consult configuring NGINX as a reverse proxy for a more general understanding.

Configure NGINX

Since your Nginx server is already set up for SSL, your configuration file has at least two servers defined and looks something like this: 
         server { listen 80;
                 server_name YourServerName or IP;
                 ...
                 # Various location contexts
                 ...
                }

        server { listen 443 ssl; 
                 server_name YourServerName or IP;
                 ssl_certificate /etc/nginx/yourcertname.crt; 
                 ssl_certificate_key /etc/nginx/yourcertname.key;
                 ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
                 ssl_ciphers HIGH:!aNULL:!MD5;
               }
  1. Change the server context listening on port 80 to redirect requests to https. 
             server { listen 80;
                 return 301 https://$host$request_uri;
                }
  2. Add a location context to the SSL server with proxying directives to send traffic to your Jenkins server 
            server { listen 443 ssl; 
                 server_name YourServerName or IP;
                 ssl_certificate /etc/nginx/yourcertname.crt; 
                 ssl_certificate_key /etc/nginx/yourcertname.key;
                 ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
                 ssl_ciphers HIGH:!aNULL:!MD5;
               }
        location / {
                    proxy_pass              JenkinsServerName or IP:8080;    
                    proxy_set_header        Host $host;
                    proxy_set_header        X-Real-IP $remote_addr;
                    proxy_set_header        X-Forwarded-For $proxy_add_x_forwarded_for;
                    proxy_set_header        X-Forwarded-Proto $scheme;
                   }
  3. The final nginx configuration file should look like this: 
               server { listen 80;
                 return 301 https://$host$request_uri;
                }

        server { listen 443 ssl; 
                 server_name JenkinsServerName or IP;
                 ssl_certificate /etc/nginx/yourcertname.crt; 
                 ssl_certificate_key /etc/nginx/yourcertname.key;
                 ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
                 ssl_ciphers HIGH:!aNULL:!MD5;

                 location / {
                    proxy_pass              JenkinsServerName or IP:8080;    
                    proxy_set_header        Host $host;
                    proxy_set_header        X-Real-IP $remote_addr;
                    proxy_set_header        X-Forwarded-For $proxy_add_x_forwarded_for;
                    proxy_set_header        X-Forwarded-Proto $scheme;
                            }
               }
  4. Restart Nginx
  5. Verify your configuration opening a browser and navigating to http://YourServerName or IP.  Your http request will change to httpsand you will be redirected to your Jenkins server.  Even though you are connected to your Jenkins server, it will appear as if YourServerName is servibg the page.  You can also access your Jenkins server by using https://YourServerName or IP.

Conclusion

You have implemented a two-server environment with an Nginx proxy with SSL and a Jenkins server.
