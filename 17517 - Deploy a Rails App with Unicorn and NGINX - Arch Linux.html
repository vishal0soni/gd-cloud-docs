Setting up Rails application with Unicorn and NGINX in Arch Linux, Ruby, Rails, NGINX

Deploy a Rails App with Unicorn and NGINX – Arch Linux

Difficulty: 2
Time: 30 minutes

In this article, we will be configuring a multi-layer deployment installation to host Rails based Ruby web applications. To do this, we will use the powerful, flexible, and extremely successful Unicorn application server running behind Nginx.

Note: We assume that you are logged in as a sudo user.

Prerequisites

In this article, we assume that you have already installed Ruby on Rails with rbenv and your Rails application uses PostgreSQL. If not, please visit:
  1. 17522-Install Ruby on Rails with rbenv – Arch Linux
  2. 17527-Use PostgreSQL with your Ruby on Rails application – Arch Linux

Prepare the deployment Server

  1. Install the bundle of development tools and update the operating system using the following command
    sudo pacman -Sy base-devel
    sudo pacman -Syu
  2. Install Nginx
    sudo pacman -S nginx
  3. Start the Nginx service
    sudo systemctl nginx start
  4. Install Unicorn and bundle all gems
    gem install unicorn
    bundle install

Setup a Rails Application

  1. Create a new directory
    sudo mkdir –p /var/www
  2. Move to the directory and provide all the required permissions
    cd /var
    sudo chmod 777 www/
  3. Create a Rails Application in this directory
    rails new HelloWorld -d postgresql
  4. Move to the newly created application directory
    cd HelloWorld/
  5. Create a user for the production database
    sudo -u postgres createuser -s username
  6. To set the database user password, run the following command
    sudo -u username psql
    \password username
  7. To drop a template, first you need to modify it
    UPDATE pg_database SET datistemplate = FALSE WHERE datname = 'template1';
  8. Now you can drop it, using
    DROP DATABASE template1;
  9. Create a database from template0, with a new default encoding
    CREATE DATABASE template1 WITH TEMPLATE = template0 ENCODING = 'UNICODE';
  10. Now modify template1 so it’s actually a template
    UPDATE pg_database SET datistemplate = TRUE WHERE datname ='template1';
  11. Now switch to template1 and VACUUM FREEZE the template
    \c template1
    VACUUM FREEZE;
  12. Start the PostgresSQL service
    sudo systemctl start postgresql
  13. Enter the new password and logout using the following command
    \q
  14. To setup a database connection, move to your application directory
    Cd /var/www/HelloWorld
    sudo vim config/database.yml
  15. Add the following line in the default section of this file
    host: localhost
  16. Notice the production section at the bottom of the file
    username: username
    password: <%= ENV[HelloWorld_DATABASE_PASSWORD'] %>
  17. If your production username and the database user are not equal, then make the required changes to set them equal
  18. Save and exit.
  19. Install the pluggin for rbenv-vars
    Cd .rbenv/plugins
    git clone https://github.com/sstephenson/rbenv-vars.git
  20. Set environments variable by generating a secret key
    cd /var/www/HelloWorld
    rake secret
  21. Copy the generated key and open the following file
    sudo vim .rbenv-vars

    Add the following line in this file

    SECRET_KEY_BASE=your_generated_secret
    HelloWorld_DATABASE_PASSWORD=prod_db_pass
  22. Save and exit.
  23. Now, export the secret key and the database password
    rbenv vars

    Export them manually if required

  24. Create the production database for the Rails application
    RAILS_ENV=production rake db:create
  25. Generate a scaffold controller to manage the production database
    rails generate scaffold Task title:string note:text
  26. Update the production database
    RAILS_ENV=production rake db:migrate
  27. Run your application from the application directory
    rails s

    To terminate user press CTRL+C.

  28. For binding to web browser, run the following command
    rails server --binding=[your droplet's IP]
  29. You should now be able to access it on any browser by
    visiting: http://[your droplet's IP]:3000/tasks
  30. Uploading source code here : 17414-Use SFTP to Securely Transfer Files with a Remote Server - ALL DISTROS

Configure Server

  1. To configure Unicorn, move to your Rails application directory
    Cd /var/www/HelloWorld
  2. Create a blank unicorn.rb document
    sudo vim config/unicorn.rb
  3. Add the following code in this file 
    	# set path to application
	app_dir = File.expand_path("../..", __FILE__)
	shared_dir = "#{Application directory}/shared"
	working_directory Application directory


	# Set unicorn options
	worker_processes 2
	preload_app true
	timeout 30

	# Set up socket location
	listen "#{shared_dir}/sockets/unicorn.sock", :backlog => 64

	# Logging
	stderr_path "#{shared_dir}/log/unicorn.stderr.log"
	stdout_path "#{shared_dir}/log/unicorn.stdout.log"

	# Set master PID location
	pid "#{shared_dir}/pids/unicorn.pid"
    Save and exit.
  4. Now create the directories that were referred in the configuration file
    sudo mkdir -p shared/pids shared/sockets shared/log
  5. Create a script for Unicorn
    sudo vi /etc/pam.d/unicorn_HelloWorld
  6. Add the following script code in the file 
    #!/bin/sh

### BEGIN INIT INFO
# Provides:          unicorn
# Required-Start:    $all
# Required-Stop:     $all
# Default-Start:     2 3 4 5
# Default-Stop:      0 1 6
# Short-Description: starts the unicorn app server
# Description:       starts unicorn using start-stop-daemon
### END INIT INFO

set -e

USAGE="Usage: $0 "

# app settings
USER="username"
APP_NAME="Application_name"
APP_ROOT="/home/$USER/$APP_NAME"
ENV="production"


# environment settings
PATH="/home/$USER/.rbenv/shims:/home/$USER/.rbenv/bin:$PATH"
#CMD="cd $APP_ROOT && bundle exec unicorn -c config/unicorn.rb -E $ENV -D"
CMD="bundle exec unicorn -c config/unicorn.rb -E $ENV -D"

PID="$APP_ROOT/shared/pids/unicorn.pid"
OLD_PID="$PID.oldbin"

# make sure the app exists
cd $APP_ROOT || exit 1

sig () {
  test -s "$PID" && kill -$1 `cat $PID`
}

oldsig () {
  test -s $OLD_PID && kill -$1 `cat $OLD_PID`
}
case $1 in
  start)
    sig 0 && echo >&2 "Already running" && exit 0
    echo "Starting $APP_NAME"
    #su - $USER -c "$CMD"
    /home/username/Application_name/bin/bundle exec unicorn -c config/unicorn.rb -E $ENV -D
    ;;
  stop)
    echo "Stopping $APP_NAME"
    sig QUIT && exit 0
    echo >&2 "Not running"
    ;;
  force-stop)
    echo "Force stopping $APP_NAME"
    sig TERM && exit 0
    echo >&2 "Not running"
    ;;
  restart|reload|upgrade)
    sig USR2 && echo "reloaded $APP_NAME" && exit 0
    echo >&2 "Couldn't reload, starting '$CMD' instead"
    $CMD
    ;;
  rotate)
    sig USR1 && echo rotated logs OK && exit 0
    echo >&2 "Couldn't rotate logs" && exit 1
    ;;
  *)
    echo >&2 $USAGE
    exit 1
    ;;
esac
  7. Save and exit.
  8. This script will allow you to run unicorn_HelloWorld that will run and stop both your unicorn and your Rails application.
  9. Update the script's permissions and enable Unicorn to start on boot
    sudo chmod 755 /etc/pam.d/unicorn_HelloWorld
  10. Start the Unicorn service
    sudo /etc/pam.d/unicorn_HelloWorld start
  11. Configure Nginx and enable it to communicate with the Unicorn. To do this, you need to edit the default configuration file: default.conf and leave nginx.conf
    sudo vim /etc/nginx/nginx.conf
  12. Swap the files contents with the following lines 
    	upstream app {
    		# Path to Unicorn SOCK file, as defined previously
   		server 	unix:/home/UserName/HelloWorld/shared/sockets/unicorn.sock 	fail_timeout=0;
	}

	server {
   	 listen 80;
   	 server_name 192.169.169.119; //your public ip

  	  root /home/username/HelloWorld/public;

   	 try_files $uri/index.html $uri @app;

   	 location @app {
        	 proxy_pass http://app;
       		 proxy_set_header X-Forwarded-For 			  		
			 $proxy_add_x_forwarded_for;
        	 proxy_set_header Host $http_host;
        	 proxy_redirect off;
    	}

   	 error_page 500 502 503 504 /500.html;
        client_max_body_size 4G;
    	 keepalive_timeout 10;
	}
    Save and exit.
  13. Restart the Nginx service
    sudo systemctl nginx start

Managing the servers

  1. Once we finish configuring both the servers, initiate the Unicorn and run it as a daemon by using the configuration file. Ensure that you are inside the application directory and run the following command
    unicorn_rails -c config/unicorn.rb –Ds
  2. Test your droplet's IP address (or the domain name associated to it) at http://[Your_droplet's_IP address]/tasks
