Install Apache Tomcat 7 - Debian, Ubuntu

Difficulty: 2
Time: 20 minutes

About Tomcat

Apache Software Foundation developed Tomcat as an open source web server to serve Java applications. This article walks through the installation and basic configuration of Tomcat7 on a Debian or Ubuntu server. It assumes you have completed the initial server setup on your Debian or Ubuntu server, and have also created a non-root user account.

Install Tomcat

There are a number of ways to install Tomcat, but we will use the apt-get method as it is the most simple. Note that this method will install the latest version of Tomcat that is in the Debian (or Ubuntu) repositories, not necessarily the latest Tomcat release.

  1. Update the packages repository.
    sudo apt-get update
  2. Install Tomcat. Type y when prompted to confirm the installation.
    sudo apt-get install tomcat7
  3. To change the default Java options which Tomcat will reference when starting, open Tomcat's parameters file.
    sudo vim /etc/default/tomcat7
  4. Replace the line that starts with JAVA_OPTS with the line below. You can also safely change the Xmx and/or MaxPermSize values to change Tomcat's memory usage.
    JAVA_OPTS="-Djava.security.egd=file:/dev/./urandom -Djava.awt.headless=true -Xmx512m -XX:MaxPermSize=256m -XX:+UseConcMarkSweepGC"
  5. Exit vim and confirm the save, if prompted.
  6. Restart Tomcat.
    sudo service tomcat7 restart

Tomcat still needs some configuration work, but you can celebrate completing the install by entering http://server_ip_address:8080 into your web browser to view the splash page. If everything was done correctly, you will see "It works!" at the top of the page.

Install additional packages

Install Tomcat's online documentation, the web application manager, and some example web apps.

  • Run the following command and type y when prompted to confirm the installations.
    sudo apt-get install tomcat7-docs tomcat7-admin tomcat7-examples

Install optional packages

If you intend to develop Java applications on this server, you will want to install the Java Development Kit (JDK).

  • To install the JDK, run the following command and type y when prompted to confirm the installation.
    sudo apt-get install default-jdk

Tomcat documentation suggests installing a source control system, like Git, as well as Apache Ant (used to develop Java applications).

  • To install Git and Ant, run the following command and type y when prompted to confirm the installations.
    sudo apt-get install git ant

Configure Tomcat's web management interface

In order to use the web application manager you installed above (under "Install additional packages"), you will first need to create a user.

  1. Open Tomcat's users file.
    sudo vim /etc/tomcat7/tomcat-users.xml
  2. Add the following line of code immediately after <tomcat-users>:
    <user username="admin" password="password" roles="manager-gui,admin-gui"/>
    You will obviously want to change the username and password values to something more secure.
  3. Exit vim and confirm the save if prompted.
  4. Restart Tomcat.
    sudo service tomcat7 restart

Access Tomcat's web management interface

Now that Tomcat knows who the admin user is, it will allow you to access the Web Application Manager, which helps you manage your Java applications. This is where you can Start, Stop, Reload, Deploy, and Undeploy applications or run diagnostics on your apps. You can find information about your server at the very bottom of the page.

  • To see the Web Application Manager, enter http://server_ip_address:8080/manager/html into your web browser.

Tomcat's Virtual Host Manager is where you add virtual hosts, which is where applications are served.

  • To see the Virtual Host Manager, enter http://server_ip_address:8080 into your web browser.
For both URLs, you will be prompted for the new user's credentials you entered above.

Conclusion

You're done! You have successfully installed and configured Apache Tomcat 7. Now go deploy your own web apps.
