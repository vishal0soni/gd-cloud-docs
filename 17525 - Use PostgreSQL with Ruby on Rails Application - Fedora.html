Setting up Rails Application with PostgreSQL in Fedora, PostgreSQL, Ruby, Rails

Use PostgreSQL with Ruby on Rails Application – Fedora

Difficulty: 2

Time: 30 mins

In this article, we'll create a PostgreSQL server to use as a database layer for scaling a Ruby on Rails application. To perform these tasks, we assume that you have sudo privileges.

To install Ruby on Rails, see Install Ruby on Rails using RVM in Fedora.

Disable SELINUX

Make sure that SELINUX is disabled.

  1. Open the following file, and disable SELINUX:
    sudo vim /etc/selinux/config
    SELINUX=disabled
  2. Save changes, and close the file.

Set up a PostgreSQL database

  1. Download the latest EPEL repositories.
    sudo wget http://dl.fedoraproject.org/pub/epel/7/x86_64/e/epel-release-7-5.noarch.rpm
    sudo rpm -ivh epel-release-7-5.noarch.rpm
  2. Set up a PostgreSQL server, and add its libraries related to development.
    sudo yum install postgresql-server postgresql-contrib postgresql-devel
    3. Initialize a new PostgreSQL database
    sudo postgresql-setup initdb
  3. Open the pg_hba.conf file so that you can modify the host-based authentication of PostgreSQL. It doesn’t allow for password authentication by default.
    sudo vim /var/lib/pgsql/data/pg_hba.conf
  4. Search for the following lines at the end of the file:
    5. host    all             all             127.0.0.1/32            ident
host    all             all             ::1/128                 ident
  5. Change "ident" to "md5”.
  6. Save and close the file.
  7. Run and enable the PostgreSQL service.
    sudo systemctl start postgresql
    sudo systemctl enable postgresql

Create a database user

  1. Log in with postgres system user.
    sudo su - postgres
  2. Create a PostgreSQL super user.
    createuser -s PostUser
  3. Set a password for the database user.
    psql
    postgres=# \password PostUser
  4. Enter the new password.
  5. Close the session.
    postgres=# \q

Set up a Rails application

  1. Move to the home directory to create a new Rails app. To set PostgreSQL as the database, use the -d postgresql option
    cd ~
    rails new HelloWorld -d postgresql
  2. Move to the application directory.
    cd HelloWorld
  3. Open the following file:
    sudo vim config/database.yml
  4. Look for "pool: 5" in the file, and add the following lines beneath it. Enter the username and password for the postgres user. 
    host: localhost
username: PostUser
password: postgres_password
  5. Save and close the file.

Create a database for the application

  • Use the rake command to create the application's databases in the application directory.
    rake db:create
    Two databases in the PostgreSQL server are created. For example, if the application's name is HelloWorld, the command creates databases named HelloWorld _development and HelloWorld _test.

Test the configuration

  • Test the Rails application.
    Rails server
    Now we can run the Rails application on our localhost on port 3000. We can also access the application from web browser if it is located on a remote server by assigning a public IP address.
    rails server --binding=server_public_IP
