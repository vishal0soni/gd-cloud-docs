Deploy Rails Apps with Passenger and Apache - Arch Linux

Difficulty: 2
Time: 30 minutes

This article guides you on how to deploy Rails Apps with Passenger and Apache in Arch Linux. You will need at least 1 GB of RAM in a virtual server to run the Passenger commands that consume a lot of memory. It is assumed that the user has sudo privileges to access the virtual server.

Note: This guide includes development library installation and ad hoc compilation on the machine, hence it is not recommended to use this tutorial in a production environment.

Prerequisites

To setup your Arch Linux server, you may refer 17324 - Initial Server Setup - Arch Linux.You will need to ensure that the FQDN points to the server. Once you have setup the Arch Linux server, you will need to 17537-Install Ruby on Rails using RVM - Arch Linux.You may also refer 17716 - Deploy Rails Apps with Passenger and NGINX - Arch Linux link.

Install Apache

Update the system and install Apache

sudo pacman -Syu
sudo pacman -S apache

Install Rails and Passenger

  1. Install Rails and Passenger
    gem install passenger
  2. Install the Passenger package for Apache
    passenger-install-apache2-module
  3. After executing this command you will see something like this at the end: 
    LoadModule passenger_module /home/Logged_in_User/.rvm/gems/Installed_ruby_Version/gems/passenger-5.0.15/buildout/apache2/mod_passenger.so
<IfModule mod_passenger.c>
PassengerRoot /home/Logged_in_User/.rvm/gems/Installed_ruby_Version/gems/passenger-5.0.15
PassengerDefaultRuby /home/Logged_in_User/.rvm/gems/Installed_ruby_Version/wrappers/ruby
</IfModule>
  4. Open the configuration file of the Apache server. Copy the above lines and paste it in the configuration file of the Apache server.
    sudo vim /etc/httpd/conf/httpd.conf
  5. Save the changes and restart the Apache service
    sudo systemctl restart httpd

Configure a test Rails application

  1. To do this, first install the sqlite's development package:
    sudo pacman -S sqlite
  2. Move the test app to Apache's web root folder
    cd /var/www/html
  3. Give the following permission.
    sudo chmod 0777 -R /var/www/html/
  4. In this folder, test application code is generated under the "HelloWorld" folder
    rails new HelloWorld
    cd HelloWorld
  5. You can also initiate the development database using the migration tool
    rake db:migrate

Connect the Apache Server to the Rails App

  1. Create a new virtual host for your application
    sudo vim /etc/httpd/conf/http.conf
  2. In this file uncomment the following line: 
    ServerName www.example.com:80 
Include conf/extra/httpd-vhosts.conf
  3. At the end of this file, add the virtual host 
    <VirtualHost *:80>
       ServerName www.mydomain.com //your server name
	 DocumentRoot /var/www/html/HelloWorld/public //your rail application path
   <Directory /var/www/html/HelloWorld/public>
      AllowOverride all
      Options -MultiViews
   </Directory>
</VirtualHost>
  4. Save and exit.
  5. Restart the Apache server
    sudo systemctl restart httpd
