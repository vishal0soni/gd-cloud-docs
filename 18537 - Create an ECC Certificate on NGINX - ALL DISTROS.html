Installing an ECC certificate on a server running NGINX.

Install ECC Certificates on NGINX - ALL DISTROS

Difficulty: 2
Time: 3 hours

Installing an Elliptic Curve Cryptography (ECC) certificate on your server can improve the security of your website, use fewer resources, and allow mobile devices to connect faster.  ECC algorithms offer higher levels of security with a smaller key size because they are based on a discrete logarithm of a random elliptic curve instead of using a large integer like RSA keys do.  From the user's perspective, the principles remain the same.  Using a combination of the certificate and a key pair, a third-party certificate authority (CA) validates the identity of both parties in a transaction and website protected by an ECC certificate is identified by a URL that starts with https.

This article walks you through installing an ECC certificate on any server running NGINX.  It assumes you have Nginx and openssl installed.

Use a self-signed certificate

Most web sites are capable of generating self-signed certificates.These are not particularly useful from a security perspective, but generating and then using one illustrates some of the steps required to install a commercial certificate.

  1. Navigate to the NGINX directory.
    cd /etc/nginx
  2. Generate your private key.
    sudo openssl ecparam -out server.key -name prime256v1 -genkey
  3. Set appropriate permissions on the key file so that it is read-only.
    sudo chmod 600 server.key
  4. Enter the metadata for certificate.
    sudo openssl req -new -key server.key -out csr.pem
    Follow the instructions that will appear on the screen. Do not supply a challenge password.  For the "Common Name" question, enter the Fully Qualified Domain Name (FQDN) of your server.  After completing the form, you will be returned to a command prompt.
  5. Sign the key and give it a shelf life of 365 days.
    sudo openssl req -x509 -days 365 -key server.key -in csr.pem -out certificate.pem
    Your screen will display: 
    Signature ok
         subject=[some of the information you just provided]
         Getting Private key

Use a commercial certificate

Using a commercial certificate is similar. The main difference is that you will follow the directions from your certificate provider to acquire your certificate and key, then move them into the /etc/nginx directory.

Set up NGINX

  1. Depending on your distribution, the ssl server context may be in the nginx configuration file or an include file.  In any event, modify / create the ssl server context so it looks like this:   
    #HTTPS server
#
server {
    listen 443;
    server_name YourServerName or IP;
    index index.html index.htm;

    ssl on;
    ssl_certificate /etc/nginx/certificate.pem;
    ssl_certificate_key /etc/nginx/server.key;

    ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
    ssl_ciphers HIGH+kEECDH+AESGCM:HIGH+kEECDH:HIGH+kEDH:HIGH:!aNULL;
    ssl_prefer_server_ciphers on;

    location / {
        try_files $uri $uri/ =404;
    }
}
  2. For a commercial certificate, edit the default.conf file as above, but with the name of your certificate instead of server. 
    server { ...
        ssl_certificate /etc/nginx/yourcertname.pem; 
        ssl_certificate_key /etc/nginx/yourcertname.key;
         ...

Set up multiples certificates on one IP

If you have multiple server blocks set up on a single IP, you can generate as many certificates as you need by following the steps above and edit each server block individually.

server { listen 443 ssl;
        server_name yourfirstserverblockname.com;
      
        ssl_certificate /etc/nginx/yourfirstserverblockcertname.crt;
        ssl_certificate_key /etc/nginx/yourfirstserverblockcertname.key;
        ...}

server_name yoursecondserverblockname.com;
      
        ssl_certificate /etc/nginx/yoursecondserverblockcertname.crt;
        ssl_certificate_key /etc/nginx/yoursecondserverblockcertname.key;
        ...}
