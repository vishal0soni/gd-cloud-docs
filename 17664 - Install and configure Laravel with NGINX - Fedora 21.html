Instructions to install and configure Laravel in a Nginx / Fedora 21 environment.

Install and configure Laravel with Nginx - Fedora 20

Difficulty: 3
Time: 20 minutes

Laravel is a framework used to create web applications. 

Update and Install Dependencies

  1. Update
    sudo yum update
  2. Install Nginx
    sudo yum install nginx
  3. Install epel
    sudo rpm -Uvh  https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
  4. Install git
    sudo yum install git
  5. Install php and related items
    sudo yum install php php-fpm php-cli php-mcrypt

Configure php

php.ini

  1. Edit php initialization file. Open /etc/php.ini for editing
    sudo vim /etc/php.ini
  2. Find the cgi.fix_pathinfo directive under a section called Paths and Directories. Modify the cgi.fix_pathinf directive as shown below:
    cgi.fix_pathinfo=0
  3. Save and close.

www.conf

  1. Edit php cofiguration file. Open www.conf for editing
    sudo vim /etc/php-fpm.d/www.conf
  2. Find the listen directive and set it as shown below:
    listen = /var/run/php5-fpm.sock
  3. Find the listen.owner and listen.group directives and set them as shown below: 
    listen.owner = nginx
listen.group = nginx
  4. Find the user and group directives and set them as shown below: 
    user = nginx
group = nginx
  5. Save and close.
  6. Restart php
    sudo systemctl start php-fpm

Configure Nginx

Nginx.conf

  1. Create a physical location for the Laravel server
    sudo mkdir -p /var/www/laravel
  2. Open the nginx configuration file for editing
    sudo vim /etc/nginx/nginx.conf
  3. Edit the /etc/nginx/nginx.conf file.  Comment out the root directive and add a new one for laravel 
    # server context
                server {
        listen       80 default_server;
        server_name  localhost;
#        root         /usr/share/nginx/html;
         root /var/www/laravel/public;

        #charset koi8-r;

        #access_log  /var/log/nginx/host.access.log  main;

        # Load configuration files for the default server block.
        include /etc/nginx/default.d/*.conf;
  4. Save and close.

php.conf

  1. Open the nginx php server block configuration file for editing
    sudo vim /etc/nginx/default.d/php.conf
  2. Edit the server and location contexts as shown below (changes / adds are bold italic): 
    # server context
index index.php index.html index.htm;


#location context
 location ~ \.php$ {
        try_files $uri /index.php =404;
        fastcgi_split_path_info ^(.+\.php)(/.+)$;
        fastcgi_pass unix:/var/run/php5-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
  3. Save and close.
  4. Restart Nginx
    sudo systemctl restart nginx

Create a swap file

In the event your server does not have sufficient memory ( > 2GB) to handle the requirements of Composer and Laravel in memory, you will probably need to create a swap file.

  1. Create a 1GB file
    sudo fallocate -l 1G /myswapfile
  2. Format it as swap space
    sudo mkswap /myswapfile
  3. Change permissions
    sudo chmod 0600 /myswapfile
  4. Turn it on
    sudo swapon /myswapfile

Install Composer

Composer is a tool we will use to manage the Laravel implementation.

  1. Navigate to your home directory and run the Composer install script.
    cd ~
    curl -sS https://getcomposer.org/installer | php
  2. You will see 
    All settings correct for using Composer
Downloading...

Composer successfully installed to: /home/randomcode/composer.phar
Use it: php composer.phar
  3. Change the name of the newly-created composer.phar file to "composer"
    sudo mv composer.phar /usr/local/bin/composer

Install Laravel

  1. Use composer to retrieve and install Laravel into the directory we created earlier
    sudo /usr/local/bin/composer create-project laravel/laravel /var/www/laravel 4.2
  2. Change the owner to "www-data".
    sudo chown -R :nginx /var/www/laravel
  3. Change the permissions to allow www-data to write to the app/storage directory
    sudo chmod -R 775 /var/www/laravel/app/storage
  4. Verify success by navigating to yourcloudserver
    http://yourcloudserver.com

Next steps

You have arrived.  Now you can use the Laravel framework to make beautiful apps.
