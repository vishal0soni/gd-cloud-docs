This article walks through installing and securing phpMyAdmin on your server.

Install and secure phpMyAdmin with Apache - Centos 6

Difficulty: 2
Time: 15 mins

phpMyAdmin is an open source tool that enables users to perform MySQL™ database operations like create, delete, or modify MySQL databases, or execute SQL statements, by providing a user interface.

Prerequisites

Make sure that you have LAMP (Linux™, Apache™, MySQL, and PHP™) installed. In case you haven't, refer to this article: Install LAMP.

Before you start

  1. Make sure that the list of available packages are up-to-date.
    sudo yum update
  2. Before you install the actual phpMyAdmin package, you will have to install the EPEL repository. phpMyAdmin is not part of the CentOS default repository. EPEL stands for Extra Packages for Enterprise Linux. It contains several packages, including phpMyAdmin.
    sudo yum install epel-release

Install phpMyAdmin

  1. Install phpMyAdmin.
    sudo yum install phpmyadmin
  2. During the installation, an Apache config file was created. We need to make some changes in this file. Open the file.
    sudo vim /etc/httpd/conf.d/phpMyAdmin.conf
  3. By default in the IfModule blocks, you will see the IP address 127.0.0.1. You will most likely find it in four places in the file. Replace the default IP address with the IP address of the computer with which you will access the application. 
        <RequireAny>
       Require ip 127.0.0.1
       [...]
       Allow from 127.0.0.1
       [...]
       Require ip 127.0.0.1
       [...]
       Allow from 127.0.0.1
       [...]

    This IP address is NOT the IP address of your server, but that of the work or home computer used to navigate to the phpmyadmin application. To find your IP address, go to http://whatismyipaddress.com/.

  4. Save and exit the file.
  5. Restart Apache.
    sudo service httpd restart
  6. On your browser, navigate to http://server-ip-address-or-your-domain-name/phpmyadmin to see the web interface. You will see a Welcome to phpMyAdmin page.
  7. Log into the user interface using the username "root" and the MySQL/MariaDB root password. You can also use any other valid MySQL/ MariaDB users. You will see the phpMyAdmin user interface.

Configure phpMyAdmin to be secure

Although your phpMyAdmin interface is up and running, you need to secure it from authorized users by adding another layer of authorization. You will do this by creating a user/password prompt.
  1. To allow the .htaccess file access the phpMyAdmin directory, you need to make some changes in the phpMyAdmin configuration file.
    sudo vim /etc/httpd/conf.d/phpMyAdmin.conf
  2. Include the AllowOverride All directive inside the <Directory /usr/share/phpmyadmin> module of the configuration file. 
    <Directory /usr/share/phpMyAdmin>
[...]
AllowOverride All
<IfModule mod_authz_core.c=mod_authz_core.c>
[.....]
  3. Save and close the file.
  4. Restart Apache for the changes to take effect.
    sudo service httpd restart

Create .htaccess file

  1. After you enable the .hataccess, it is time to create the actual .htaccess file. This file will be inside the application directory.
    sudo vim /usr/share/phpMyAdmin/.htaccess
  2. Enter the block of text below into the file: 
    AuthType Basic
AuthName "Restricted. Only authorized users permitted."
AuthUserFile /etc/httpd/pass_file
Require valid-user
  3. Below is an explanation of what each of these directives mean :
    Directive Description
    AuthType Basic Specifies the kind of authentication that is being implemented. "Basic" implies that it uses a password file to implement the password authentication.
    AuthName This is the message that is displayed at the password prompt. You can set it to anything you want.
    AuthUserFile The location where you would store the password file.
    Require valid-userIndicates that only users who are stored in the password file will be granted access to the phpMyAdmin login screen.
  4. Save and close the file.

Create user

  1. Create a first user to be stored in the password file.
    sudo htpasswd -c /etc/httpd/pass_file newuser1
  2. Set and confirm password.
  3. If you want to enter an additional user, you need to do so without the -c flag.
    sudo htpasswd /etc/httpd/pass_file newuser2
  4. Restart Apache for changes to take effect.
    sudo service httpd restart
  5. Now, when you navigate to the browser and type http://server-ip-address-or-your-domain-name/phpmyadmin, you will be prompted for the username and password. Enter the newuser1 credentials that you included in the htpasswd file.
