Setting up Rails Application with PostgreSQL in Ubuntu, PostgreSQL, Ruby, Rails

Difficulty: 2

In this article, we'll create a PostgreSQL server to use as a database layer for scaling a Ruby on Rails application. To perform these tasks, we assume that you have sudo privileges.

To install Ruby on Rails, see Install Ruby on Rails using RVM in Ubuntu.

Install PostgreSQL

  1. At the terminal, type the following command:
    sudo apt-get install postgresql postgresql-contrib
  2. Next install pgadmin3 for the administrator. pgadmin3 is an easy-to-access GUI for PostgresSQL. It's an optional installation but useful when used on a desktop.
    sudo apt-get install pgadmin3

Create database user

  1. Log in with postgres system user, and create a user.
    sudo -u postgres createuser -s PostUser
  2. Set a password for the database user.
    sudo -u postgres psql
    postgres=# \password PostUser
  3. Close the session.
    postgres=# \q
  4. Restart the PostgresSQL service.
    sudo service postgresql restart

Set up the Rails application

  1. Move to the home directory to create a new Rails app. To set PostgreSQL as the database, use the -d postgresql option.
    cd ~
    rails new HelloWorld -d postgresql
  2. Move to the application directory.
    cd HelloWorld
  3. Open the following file.
    sudo vim config/database.yml
  4. Look for "pool: 5" in the file, and add the following lines beneath it. Enter the username and password for the postgres user. 
    host: localhost
username: PostUser
password: postgres_password
  5. Save and close the file.

Create a database for the application

  • Use the rake command to create the application's database in the application directory.
    rake db:create
    Two databases in PostgreSQL server are created. For example, if the application's name is HelloWorld, the command creates databases named HelloWorld _development and HelloWorld _test.

Test the configuration

  • Test the Rails application.
    Rails server
    Now we can run the Rails application on our localhost on port 3000. We can also access the application from a web browser if it is located on a remote server by assigning a public IP address.
    rails server --binding=server_public_IP
